# Handoff: reasoning-model summaries, query-embedding cache, auto-unmount, short history

Owner report (2026-07-18), four items on the AI & Models / summary flow:

1. Switching the embedding model in Search is slow on the first query, then fast — the app re-embeds
   the query each time (a network round-trip for an Ollama embedder, plus the first-call model load).
   Wanted: a per-model LRU cache for query embeddings, size configurable (default 2048).
2. Summaries got *worse* after switching qwen3 → qwen3.5 (both 4b): output started "mid-thought",
   badly formatted, and the card showed a self-contradicting note — "Summarized with the extractive
   fallback (LLM unavailable). Generated by qwen3.5:4b". The LLM was clearly running on the GPU, then
   extractive ran and its text was shown but labeled as the LLM's.
3. No summary "history" for short summaries (only detailed had it).
4. Wanted "auto-unmount" + "auto-unmount time" as changeable settings.

## Root causes & fixes

### 2 — reasoning models (THE main bug)
`_ollama_generate` (summarization.py) sent no `think` flag and never stripped `<think>…</think>`.
qwen3/qwen3.5/deepseek-r1 are **reasoning models**: they emit a long chain-of-thought first. That
(a) leaked into the summary ("mid-thought"), and (b) the slow reasoning blew the hard 120 s timeout →
exception → **extractive fallback**. So the LLM spun the GPU, then extractive produced the shown text.
Fix: send `"think": false` (unknown to old daemons → harmlessly ignored) **and** strip any leaked
`<think>` block (`_strip_reasoning`) as a safety net. Same `think:false` added to
`recommendation.ollama_generate_json`. **Verified live**: qwen3.5:4b now answers a summary prompt in
**2.8 s** with clean prose and no `<think>` — no fallback.

Provenance contradiction: on fallback the row keeps the *requested* model in `model_name` (deliberate
for the scope read path; per-work reads list-all so it's display-only). Fixed in the UI: a new
`provenanceName(s)` in `WorkDetail.svelte` shows `provider_used` (e.g. "extractive") when
`s.fallback`, so the "Generated by …" line no longer contradicts the fallback note.

### 1 — query-embedding cache
`embed_query(provider, query, *, cache_size)` in `embeddings.py`, backed by `BoundedTTLCache`, keyed
by `(provider.model_name, query)` (model_name encodes provider+model+tag → per-model isolation), 24 h
TTL, rebuilt when the size changes, 0 disables. Wired at the two query→vector sites
(`chunk_search.py`, `semantic_search.py`). Cleared in `evict_cached_providers` so a re-pulled model
can't serve a stale vector. Size = new `query_cache_size` config (default 2048).

### 4 — auto-unmount
New config `auto_unmount` (bool, default true) + `auto_unmount_minutes` (default 5). `keep_alive_value(cfg)`
returns Ollama `keep_alive` seconds: `-1` (pin, never auto-unload) when off, else `minutes*60`.
Threaded into **every** on-demand Ollama call — all summary generate calls (work + scope), and the
Ollama embedder's `embed`/`embed_many` (refreshed on each `resolve_embedding_provider`). Default (on,
5 min) reproduces Ollama's built-in behavior exactly, so no regression; turning it off keeps models
resident until manually unmounted. Manually **Mounted** models are pinned independently and unaffected.

### 3 — short-summary history
`WorkDetail.svelte`: added `shortHistory` (non-detailed rows beyond the one shown) + a History (N)
popup mirroring the detailed one. Short rows already accumulate one per model (switching the summary
model creates a new row), so the history is populated.

## Config plumbing (all three new fields)
`Settings` defaults (`query_cache_size`, `ai_auto_unmount`, `ai_auto_unmount_minutes`) → `AIConfig`
columns → **alembic 0081** → `ai_config.py` (EDITABLE_FIELDS, EffectiveAIConfig, `_defaults`,
`_validate`, `keep_alive_value`) → `AIConfigUpdate` → `client.ts` `AiConfig` → `AiModelsPanel.svelte`
inputs (Query cache, Auto-unmount toggle + minutes) + Help-dialog copy.

**Falsy-overlay fix**: the generic get/persist loops used `if value:` / `changes[field] or None`,
which would drop a legitimate `False`/`0`. Reworked to treat empty strings as "unset" but keep falsy
bool/number values for `_FALSY_VALID_FIELDS` (so auto_unmount off / cache size 0 persist correctly).

## Notes / follow-ups
- `keep_alive` was **not** threaded into `recommendation.ollama_generate_json` (only `think:false`),
  so the recommendation ranker still uses Ollama's default TTL — out of scope; revisit if auto-unmount
  should also govern it.
- Scope-summary fallback still stores the requested model name by design (its read path keys on it);
  the UI `provenanceName` fix covers both work and scope displays.
