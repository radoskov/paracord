## Feature 1 — multi-PDF staging import ✅

- **Files changed:**
  - `backend/app/models/import_staging.py` (+ registered in `models/__init__.py`); migration
    `backend/alembic/versions/0056_import_staging.py`.
  - `backend/app/services/import_staging.py` — `stage_pdfs`, `extract_staging_item` (record-free,
    injected `fetch_tei`), `detect_collisions` (same_pdf/same_doi/same_title), `finalize_if_ready`,
    `auto_decisions` (direct mode), `commit_staging` (per-item SAVEPOINT so a DOI clash skips just
    that item; applies stored TEI via `store_parsed_extraction`), `enqueue_post_commit_jobs`.
  - `backend/app/services/storage.py` — public `stage_managed_file` (content-address, no Work).
  - `backend/app/workers/queue.py` — `STAGE_EXTRACT_JOB` + `enqueue_staging_extraction`.
  - `backend/app/workers/jobs.py` — `extract_staging_item_job` (extract one item; on the last item
    of a `direct` batch, auto-commit as the batch creator).
  - `backend/app/api/v1/endpoints/imports.py` — `POST /upload-multi`, `GET /staging/{id}`,
    `POST /staging/{id}/commit` + schemas. Enqueues per-item extraction; inline fallback when the
    queue is unavailable so the flow always completes.
  - `backend/openapi.json` regenerated.
  - `frontend/src/api/client.ts` — `StagingBatch`/`StagingItem`/`StagingCommitResult` types +
    `uploadPdfsMulti`/`getStagingBatch`/`commitStagingBatch`.
  - `frontend/src/pages/ImportPage.svelte` — the PDF card now takes multiple files with
    "Preview & choose" / "Import directly", a preview table (title/authors/year/collisions/status +
    per-row accept), commit, and a result summary. (Removed the old single-upload card; `uploadPdf`
    client method retained.)
- **Design decisions (owner-approved):** extract-first via a staging model; direct mode blocks on
  same-PDF **or** same-DOI (fuzzy title is a non-blocking warning). Extraction is async on the worker
  with an inline fallback (keeps dev/tests and Redis-down deployments working).
- **Tests:** `backend/tests/test_multi_import_staging.py` (stage→extract→commit, collisions,
  auto-decisions, HTTP preview+commit, direct auto-commit, direct same-DOI skip);
  `frontend/src/pages/ImportPage.test.ts` (preview commit, direct summary, blocked item unchecked).
  Migration parity (autogenerate no-drift) + `test_table_presence` green.
- **Security:** upload-multi is EDITOR+; staging batches are owner/admin-scoped; commit is
  CONTRIBUTOR+ and ACL-checks the target shelf. No arbitrary filesystem path is accepted.
- **Known limitations / next:** extraction is inline-fallback synchronous when the queue is down (a
  large batch then holds the request); consider a staging-batch cleanup/expiry sweep for abandoned
  previews; fuzzy-title collision is exact-normalized-title only (no ratio match yet).
