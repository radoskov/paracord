## Feature 4 — editable authors in Details ✅

- **Files changed:**
  - `backend/app/api/v1/endpoints/works.py` — new `SetMetadataValue` model + `POST
    /{work_id}/metadata/set` (`set_metadata_value`): writes a `source="user"` canonical assertion,
    locks the field (`confirmed_fields`), applies to the Work column for promotable fields, empty
    value clears + unlocks. Emits `metadata.value_set` audit event. Returns the refreshed
    `list[FieldReview]`.
  - `frontend/src/api/client.ts` — `setMetadataValue(workId, fieldName, value)`.
  - `frontend/src/components/WorkDetail.svelte` — `authorsFromFields()` helper (shared with
    `searchedAuthors`); `form.authors` seeded after metadata load; editable Authors input in the
    Details panel; `save()` calls `setMetadataValue` only when authors changed.
  - `backend/openapi.json` regenerated.
- **Assumptions:** authors is a `;`-separated free-text string (no structured author model).
- **Tests:** `test_metadata_assertion_delete.py` — set creates locked user assertion, promotable
  field updates Work, empty clears+unlocks. `WorkDetail.metadata.test.ts` — seeds + persists change,
  no-op when unchanged. Both green.
- **Security:** modify-guard on the endpoint; manual value locked so enrichment won't clobber it.
